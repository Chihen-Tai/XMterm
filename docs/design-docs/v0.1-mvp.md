# XMterm v0.1 MVP

- **Status:** Draft
- **Owner:** Project owner
- **Scope:** First usable terminal + remote file + local editor synchronization
  release

**Implementation note (Phase 2):** one fixed relay terminal now uses system
OpenSSH through the Phase 1 PTY with honest process-only state and conservative
live-close confirmation. The broader v0.1 profile, reconnect, SFTP, file, and
editor-sync criteria below remain draft and are not implied by that slice.

## User story

As a macOS SSH user, I can connect with my existing key, keep several terminal tabs
open, use normal terminal selection/copy/paste, browse and manipulate remote files
beside the terminal, edit selected remote files in local VS Code, and have saves
uploaded without starting a remote IDE server.

## Release rule

The release is not accepted if it only proves SSH/SFTP connectivity. All applicable
ordinary desktop interactions in `INTERACTIONS.md` must also work. Use
`docs/checklists/interaction-parity.md` during review. Terminal work must also pass
`docs/checklists/terminal-acceptance.md`.

## Acceptance criteria

### Terminal and terminal tabs

- `TAB-001`: `Command-T` and the `+` button create a terminal for the selected
  session.
- `TAB-002`: tabs can be selected by click/shortcut and reordered by drag.
- `TAB-003`: `Command-W`, close button, and middle-click close only the intended
  terminal; local PTYs close immediately when their shell is foreground and confirm
  only a different foreground group or an exceptional live query failure. Connected
  SSH tabs retain an honest conservative confirmation without pretending XMterm can
  detect remote foreground jobs perfectly.
- `TAB-004`: tab context actions include Rename, Duplicate, Reconnect, Close Others,
  and Close Tabs to the Right.
- `TERM-SEL-001` through `TERM-SEL-004`: drag, word, line, extended,
  Option-forced local selection, wrapped-line copy, and drag export work.
- `TERM-CLIP-001` through `TERM-CLIP-003`: copy, paste, and terminal context menu
  behave correctly.
- `TERM-SCROLL-001`: scrollback does not jump while reading older output.
- `TERM-FIND-001`: `Command-F` searches terminal history locally.
- `TERM-KEY-001` through `TERM-KEY-004`: terminal bytes, special keys, Unicode
  input, repeat behavior, and app Command shortcuts are distinguished correctly.
- `TERM-RENDER-001`, `TERM-RENDER-002`, and `TERM-RESIZE-001`: the advertised
  xterm-compatible rendering, Unicode width, resize, and reflow behavior pass the
  terminal acceptance checklist.
- `TERM-LINK-001`, `TERM-BELL-001`, and `TERM-SEC-001`: links require user action,
  bell follows settings, and remote clipboard access is permission-gated.
- `TERM-STATE-001`: exit/disconnect preserves scrollback and reconnect starts a
  fresh process honestly.
- Each tab has independent process state and can reconnect independently.
- SSH uses the selected `~/.ssh/config` alias.

### Remote files

- The sidebar follows the selected terminal session.
- `FILE-SEL-001`: single, Command multi-selection, Shift range selection,
  keyboard selection, and Select All work.
- `FILE-NAV-001`: double-click, Return rename, Back/Forward/Up, path entry, and
  selection restoration work.
- `FILE-CLIP-001`: Copy, Cut, and Paste work for one or many selected entries.
- `FILE-DND-001` and `FILE-DND-002`: remote move/copy drag and Finder upload/download
  drag work with clear feedback.
- `FILE-OPS-001`: required context/toolbar file actions are available.
- `FILE-OPS-002`: name collisions never overwrite silently.
- `FILE-OPS-003`: remote delete is explicit, confirmed, and batch-aware.
- `FILE-LIST-001`: listing metadata and local sorting work.
- `FILE-PERF-001`: expanding a directory lists only that directory.
- `FILE-META-001` and `FILE-XFER-001` through `FILE-XFER-004`: permissions,
  executable bits, symlinks, per-item progress, safe staging, retry ordering, and
  exact structured paths are handled.
- Operations are cancellable and display exact remote paths on destructive actions.

### Local editor sync

- `EDIT-001`: double-clicking a supported text file downloads it into a
  session-scoped cache and opens the configured local editor.
- `EDIT-002`: normal and atomic editor saves are detected reliably.
- `EDIT-003`: each logical save triggers safe debounced upload with visible state.
- `EDIT-004`: changed remote metadata creates a conflict state instead of silent
  overwrite.
- `EDIT-005`: multiple files can be watched and uploaded independently.
- `EDIT-006`: quitting with unsynced changes warns the user and cache mappings can
  recover after restart.
- `EDIT-007`: a complete revision is finalized safely, preserves remote mode, and
  remains Pending Upload while offline until conflict metadata is refreshed.

### Session and authentication

- `SESS-001`: SSH aliases are discovered and only non-secret UI metadata is stored.
- `SESS-002`: session selection, Open Terminal, Open Another Terminal, and file
  browser connection actions work.
- `SESS-003`: trusted host-key and interactive authentication prompts remain
  understandable; passwords and OTPs are never persisted by XMterm.
- `SESS-004`: effective SSH configuration is delegated to system OpenSSH.
- `SESS-005`: sleep/wake, network loss, cancellation, and reconnect have explicit
  states.
- `SESS-006`: terminal and SFTP avoid duplicate authentication when safe connection
  reuse is available, without coupling their failures.

### Resource and quality behavior

- No process named for XMterm is installed on the remote host.
- No recursive remote indexing occurs.
- Closing one terminal does not close other terminals or the application.
- A stalled transfer does not block terminal interaction.
- `APP-001` through `APP-008` apply across the app.
- `A11Y-001` through `A11Y-003` apply to primary workflows.
- `MAC-001` through `MAC-006` apply to menus, focus, close/quit, restoration,
  settings, path drag, diagnostics, and notifications.

## Explicitly deferred

- tmux integration;
- split terminal panes;
- embedded editor;
- project-wide remote search/indexing;
- X11/RDP/VNC;
- cross-platform builds.

## Open questions

- Should the initial SFTP backend wrap OpenSSH batch mode or use a native library
  immediately?
- Which local-file event API best handles atomic saves across supported editors?
- Which AppKit collection view best provides native remote-file multi-selection and
  drag/drop without compromising lazy loading?
- How should XMterm share authentication between terminal and SFTP while preserving
  OpenSSH compatibility and independent failure recovery?
- What sandbox, entitlement, signing, notarization, and distribution strategy allows
  PTY, OpenSSH, `~/.ssh`, local editor launch, file watching, and Finder drag promises
  without weakening security?

## Resolved foundations

- ADR 0003 selects one exactly pinned native engine, SwiftTerm 1.14.0, behind an
  XMterm-owned `forkpty`/`execve` process boundary. Phase 1 evidence and honest
  compatibility exceptions are recorded in Audit 0002.
